WHY ASROCK RACK (vs consumer boards)
--------------------------------------------------------------------------------

1. IPMI/BMC INCLUDED - Remote management even when OS is dead
2. PROPER ECC SUPPORT - Data integrity for server workloads
3. SERVER-GRADE COMPONENTS - Designed for 24/7 operation
4. CHEAPER - B650D4U at $299 vs X670E Taichi at $480
5. BETTER FOR HEADLESS - No wasted GPU outputs, focused on server features

Consumer boards (ASUS ROG, Gigabyte AORUS, etc.):
- No IPMI - need physical access or separate KVM-over-IP device
- Most don't support ECC properly (even if they claim to)
- Designed for desktop/gaming, not 24/7 server operation
- Pay extra for RGB, audio, features you won't use headless

PROXMOX SETUP FOR CLAUDE CODE VM FARM
--------------------------------------------------------------------------------

1. INSTALL
   - Install Proxmox VE on one 990 Pro NVMe
   - Mirror to second 990 Pro for boot redundancy (optional)

2. STORAGE POOLS
   - NVMe: ZFS mirror for VM disks (fast I/O)
   - HDDs: ZFS raidz1 or raidz2 for bulk storage, ISOs, backups

3. NETWORK
   - Configure IPMI on separate management VLAN (security)
   - Main NICs for VM traffic

4. VM TEMPLATES FOR CLAUDE CODE
   - Ubuntu 22.04/24.04 LTS template (general development)
   - ESP-IDF development template (pre-installed toolchain)
   - Analytical workload template (Python, Jupyter, etc.)
   - Clone templates on-demand for tasks

5. AUTOMATION
   - Proxmox API for VM creation/destruction
   - Cloud-init for automatic VM configuration
   - SSH keys for Claude Code access

6. BIOS SETTINGS
   - Enable IOMMU (AMD-Vi) for PCIe passthrough
   - Enable SVM Mode for virtualization
   - Set power limit to 105W for quieter operation (optional)
